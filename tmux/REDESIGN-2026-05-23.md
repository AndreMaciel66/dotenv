# Redesign do tmux — 2026-05-23

Objetivo: tmux visualmente coerente com o prompt do zsh (mesma paleta,
mesmo "vocabulário" de cores) e remover ruído visual sem adicionar plugin.
Zero TPM, zero `catppuccin/tmux`, só `tmux.conf` + 5 scripts.

> **Nota de paleta:** TMUX e Nvim usam **Catppuccin Mocha**.
> Ghostty mudou pra **Catppuccin Frappe** depois deste redesign (decisão
> intencional do user em 2026-05-23 pra distinguir contextos visualmente —
> o terminal "host" tem tom diferente do que roda dentro dele).

## O que mudou (resumo)

### `.tmux.conf`

- **`status-position top`** (libera a linha de baixo pro prompt do zsh).
- **Variáveis `%hidden`** no topo pras cores Catppuccin Mocha — trocar
  paleta vira 1 linha de edição.
- **status-left:** sessão em mauve, sem bloco cheio — coerente com `%~`
  do prompt zsh.
- **Indicadores condicionais:** `PFX` em vermelho quando o prefix está
  armado, `CPY` em amarelo no copy-mode. Somem sozinhos.
- **Window-status:** `▎` azul sutil no current em vez de bloco cheio;
  separador (`•`) discreto pra atividade em janelas inativas.
- **`pane-border-status top`** com `#P · cmd` minimalista.
- **`monitor-activity on` + `visual-activity off`** — atividade gera marker
  visual sem flash/som.
- **`bind C-p` (capture-pane) bug fix:** glob `/tmp/tmux-capture-*.txt`
  abria TODOS os capturas antigos; agora usa só o arquivo novo.
- **`status-right`:** hora (`%H:%M`) movida pra FORA do `#(...)` —
  versões novas do tmux não re-interpretam formats dentro do output de
  shell-commands, então `%H:%M` no script aparecia literal.

### Copy mode (yank `y` + mouse drag-end)

- **`copy-pipe-no-clear`** em vez de `copy-pipe-and-cancel` — não pula
  mais pro final do scrollback ao copiar.
- **Flash verde de ~180ms na seleção** como feedback ("copiei!"). Implementado
  via `set -g mode-style green` + `run-shell -b 'sleep 0.18; tmux set -g
  mode-style blue'`.
- **Branches Wayland/X11 removidos** — host é macOS-only. `set-clipboard
  on` no topo continua dando OSC52 sobre SSH.

### `scripts/status-right.sh`

Antes: 4–5 invocações de `git`, cada com `timeout 2`. Em monorepo a barra
ficava 6–8s travada.

Depois: **1 invocação** — `git status --porcelain=v2 --branch`. Parse
local com `case` num `while-read`. Output espelha o prompt zsh:

```
 venv    main ●    proj
```

- branch em **green** (`#a6e3a1`) — mesma cor do branch no prompt zsh
- `●` em **yellow** (`#f9e2af`) — mesmo símbolo/cor do `dirty` no zsh
- basename do dir em **mauve** (`#cba6f7`) — mesma cor que `%~` no prompt
- hora em **overlay0** — inserida diretamente pelo tmux (não pelo script),
  ver bug do `%H:%M` acima.

### `scripts/cheatsheet.sh`

Reescrito do zero. Antes era um menu interativo multi-página estilo TUI
(Alacritty/Tmux/Claude/Vim, com fzf search). Tentamos centralizar num
popup horizontal 95% — não saiu bem visualmente.

**Layout final (estilo `which-key.nvim`):**
- Popup colado à **esquerda da tela** (`-x 0 -y 0`), largura 52 cols,
  altura 100%.
- Conteúdo **vertical**, scroll com `less -R -X -F -i -K`.
- **Navegação 100% vi-like** (do próprio less): `j/k`, `d/u`, `f/b`,
  `gg/G`, `/search`, `?back`, `n/N`, `q` pra sair.
- Sem multi-página — TMUX → NVIM → GHOSTTY → CLAUDE numa lista única.
- Nvim keymaps lidos do `init.lua` real (não fictícios como antes).

Binding: `prefix + h`.

### `scripts/fzf-open.sh` + `fzf-open-robust.sh`

`vim --` hardcoded → `${EDITOR:-nvim}`. O `$EDITOR` do usuário já é nvim.

## Como testar

```sh
# 1. Recarregar (sessão existente)
tmux source-file ~/.tmux.conf      # ou prefix + r

# 2. Verificar visual
#    - Status no topo, sessão em mauve à esquerda
#    - ▎ azul na janela atual
#    - À direita: ● amarelo se dirty, basename mauve, hora dim
#    - prefix armado → "PFX" vermelho aparece
#    - copy-mode (prefix + Esc) → "CPY" amarelo

# 3. Copy + flash
#    - prefix + Esc, v pra selecionar, y pra copiar
#    - Seleção deve piscar VERDE por ~180ms (em vez de pular pro fim)
#    - Mouse drag-end também copia + pisca

# 4. Cheatsheet
#    - prefix + h → popup à ESQUERDA, vertical
#    - j/k pra scrollar, /termo pra buscar, q pra sair

# 5. Bench do status-right
time bash ~/.config/tmux/scripts/status-right.sh ~/algum/repo/grande
# Esperado: < 50ms em repos normais, < 250ms no pior caso
```

## Tradeoffs

- **`status-position top`**: incomum (default tmux é bottom). Trocar
  com 1 linha: `set -g status-position bottom`.
- **`pane-border-status top`**: ocupa 1 linha por pane. Em setups com
  1 pane é desperdício. Desligar: `set -g pane-border-status off`.
- **`monitor-activity on`**: pode incomodar com `tail -f`/`watch` rodando.
  Desligar: `setw -g monitor-activity off`.
- **`copy-pipe-no-clear`** mantém a seleção visível pós-copy. Se preferir
  que ela suma: trocar pra `copy-pipe` (mantém modo, limpa seleção).
- **Cheatsheet vertical**: ocupa 52 cols da esquerda. Em telas pequenas
  pode atrapalhar. Ajustar largura no `bind h display-popup -w 52 ...`.
